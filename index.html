延迟效应定量分析

1. 背景与概念

在网络同步的实时应用中，由于数据传输存在固有延迟，不同客户端对同一事件的感知会出现差异。本文旨在定量分析这种延迟对双方交互（如攻击判定）产生的影响。

1.1 感知的定义

对于任意实体或现象 σ\sigmaσ，在客户端 xxx 的感知下记作 σx^\sigma^{\hat{x}}σx^。

1.2 符号说明

2. 基本假设

为简化分析，作出以下关键假设：

  • 假设1：服务器内部处理延迟为零，所有延迟仅来源于网络传输。
  • 假设2：位置信息在客户端与服务器之间按时间顺序可靠传输。
  • 假设3：每个客户端对自身位置的感知是即时的，即 px(t)=Pxx^(t)p_x(t) = P_x^{\hat{x}}(t)px​(t)=Pxx^​(t)。

3. 延迟下的感知推导

3.1 关键参数定义

定义半程时延 rrr 和全时延 RRR：

3.2 感知位置的基本关系

客户端 A 发出的位置信息需要经过时间 rrr 才能被 B 完全接收。因此，双方对彼此位置的感知满足：

3.3 感知距离的表达

在任意时刻 ttt，双方感知到的彼此距离为：

    DB^(t)=PBB^(t)−PAB^(t)=pB(t)−pA(t−r) \begin{aligned} D^{\hat{B}}(t) &= P_B^{\hat{B}}(t) - P_A^{\hat{B}}(t) \\ &= p_B(t) - p_A(t - r) \end{aligned} DB^(t)​=PBB^​(t)−PAB^​(t)=pB​(t)−pA​(t−r)​

  • 在 A 的感知中：

3.4 感知距离的增量

将以上两式结合，可得双方感知距离的关系：

即：

DB^(t)=DA^(t)+ΔrpB(t)+ΔrpA(t) D^{\hat{B}}(t) = D^{\hat{A}}(t) + \Delta_r p_B(t) + \Delta_r p_A(t) DB^(t)=DA^(t)+Δr​pB​(t)+Δr​pA​(t)

    ∃t0,DA^(t0)<3 \exist t_0,D^{\hat{A}}(t_0) < 3 ∃t0​,DA^(t0​)<3

∃tx,3<DB^(t0+tx)≤DB^(t) \exist t_x, 3 < D^{\hat{B}}(t_0 + t_x) \leq D^{\hat{B}}(t) ∃tx​,3<DB^(t0​+tx​)≤DB^(t)

4.3 结论

A成功发动Combo需同时满足：

∃t0,tx,3−Γ(t0,tx)<DA^(t0)<3 \boxed{\exist t_0, t_x, \quad 3 - \Gamma(t_0, t_x) < D^{\hat{A}}(t_0) < 3} ∃t0​,tx​,3−Γ(t0​,tx​)<DA^(t0​)<3​

其中，我们称 Γ(t0,tx)\Gamma(t_0, t_x)Γ(t0​,tx​) 为 combo 容错，定义如下：

Γ(t0,tx)=[ΔtxpB(t0+tx−r)+ΔrpB(t0+tx)]+[ΔrpA(t0+tx)−ΔtxpA(t0+tx)] \Gamma(t_0, t_x) = [\Delta_{t_x} p_B(t_0 + t_x - r) + \Delta_r p_B(t_0 + t_x)] + [ \Delta_r p_A(t_0 + t_x) - \Delta_{t_x} p_A(t_0 + t_x)] Γ(t0​,tx​)=[Δtx​​pB​(t0​+tx​−r)+Δr​pB​(t0​+tx​)]+[Δr​pA​(t0​+tx​)−Δtx​​pA​(t0​+tx​)]

A 感知形式:

Γ(t0,tx)=[ΔtxPBA^(t0+tx)+ΔrPBA^(t0+tx+r)]+[ΔrpA(t0+tx)−ΔtxpA(t0+tx)] \boxed{\Gamma(t_0, t_x) = [\Delta_{t_x} P_B^{\hat{A}}(t_0 + t_x) + \Delta_r P_B^{\hat{A}}(t_0 + t_x + r)] + [ \Delta_r p_A(t_0 + t_x) - \Delta_{t_x} p_A(t_0 + t_x)]} Γ(t0​,tx​)=[Δtx​​PBA^​(t0​+tx​)+Δr​PBA^​(t0​+tx​+r)]+[Δr​pA​(t0​+tx​)−Δtx​​pA​(t0​+tx​)]​

这里 txt_xtx​ 是满足 DB^(t0+tx)≤DB^(t)D^{\hat{B}}(t_0 + t_x) \leq D^{\hat{B}}(t)DB^(t0​+tx​)≤DB^(t)（即B感知距离最小值时刻）的时间偏移。

4.4 txt_xtx​ 的取值

txt_xtx​ 的本质，是B感知距离最近，且可以攻击时候，与 A 攻击时 t0t_0t0​ 的偏移量。

4.4.1 无间隔对刀

Γ(t0,r)=ΔRPBA^(t0+R) \boxed{\Gamma(t_0, r) = \Delta_R P_B^{\hat{A}}(t_0 + R)} Γ(t0​,r)=ΔR​PBA^​(t0​+R)​

5. 核心结论

一、延迟效应的本质：感知分裂

由于网络存在固定的往返时延（R=τA+τBR = \tau_A + \tau_BR=τA​+τB​），双方客户端对游戏世界的感知并不同步，且存在固定的时间差。

  • 核心参数：半程时延 r=R/2r = R/2r=R/2。
  • 感知关系：一方接收到对方位置信息时，该信息已是 rrr 时间前的旧状态。这导致双方计算的实时距离不同。
  • 距离公式：在任意时刻 ttt，B感知到的距离比A感知到的距离多出一项“位移和”：DB^(t)=DA^(t)+ΔrpB(t)+ΔrpA(t) D^{\hat{B}}(t) = D^{\hat{A}}(t) + \Delta_r p_B(t) + \Delta_r p_A(t) DB^(t)=DA^(t)+Δr​pB​(t)+Δr​pA​(t) 这意味着，延迟放大了双方因移动而产生的距离偏差。

二、combo的判定条件

A能持续击中B而B无法反击的数学本质，是双方感知距离同时满足两个矛盾的条件：

  1. 在A的感知中，距离进入攻击范围（DA^<3D^{\hat{A}} < 3DA^<3）。
  2. 在B的感知中，距离始终超出攻击范围（DB^>3D^{\hat{B}} > 3DB^>3）。

将两个条件结合，推导出A能发动Combo的可操作区间：

∃t0,tx,3−Γ(t0,tx)<DA^(t0)<3 \boxed{\exist t_0, t_x, \quad 3 - \Gamma(t_0, t_x) < D^{\hat{A}}(t_0) < 3} ∃t0​,tx​,3−Γ(t0​,tx​)<DA^(t0​)<3​
  • Γ(t0,tx)\Gamma(t_0, t_x)Γ(t0​,tx​)：Combo容错。这是一个由双方位移量构成的正值。
  • 核心洞见：A不需要在自己的感知中“非常贴近”B（即 DA^D^{\hat{A}}DA^ 不需要远小于3）。只要 DA^D^{\hat{A}}DA^ 落在区间 (3−Γ,3)(3-\Gamma, 3)(3−Γ,3) 内，A就能利用延迟造成的感知差，实现“我打得到你，你打不到我”。

三、两种典型场景下的容错 Γ\GammaΓ

  1. 无间隔对刀（tx=0t_x = 0tx​=0）：

    Γ=ΔrPBA^(t0+r)+ΔrpA(t0) \Gamma = \Delta_r P_B^{\hat{A}}(t_0 + r) + \Delta_r p_A(t_0) Γ=Δr​PBA^​(t0​+r)+Δr​pA​(t0​)

    解读：容错取决于对方在A自身未来感知中的位移与A自身当前位移之和。若对面的速度比自己的小（互相面朝的方向为正），Combo更易触发。

  2. 维持combo，对方空刀，hitselect（tx=rt_x = rtx​=r）：

    Γ=ΔRPBA^(t0+R) \Gamma = \Delta_R P_B^{\hat{A}}(t_0 + R) Γ=ΔR​PBA^​(t0​+R)

    解读：容错简化为对方在A的感知中，跨越整个延迟周期 RRR 的总位移。当B被击中后处于受击飞状态时，其位移 PBA^P_B^{\hat{A}}PBA^​ 相对固定且可预测，使得A可以稳定维持连击。

四、整体结论与意义

  1. 延迟创造非对称优势：网络延迟并非只带来负面体验。精通延迟机制的玩家可以获得系统性优势，能在对方感知的安全距离外发起有效攻击。
  2. Combo的可行性窗口：攻击成功的关键不是绝对位置，而是感知差异。存在一个由延迟和双方运动状态共同决定的“容错窗口”（Γ\GammaΓ）。开发者平衡游戏或玩家提升技巧时，都必须考虑此窗口。
  3. 定量分析的价值：该模型将“感觉”层面的延迟优势，转化为可计算的位移与时间参数（r,R,Δpr, R, \Delta pr,R,Δp），为优化网络代码、设计公平机制或制定高级对战策略提供了理论依据。

总之，延迟效应打破了客户端间的感知一致性，其本质是“时间差”。而攻击判定的分裂，是这一时间差被双方运动状态放大后的具体表现。通过计算容错 Γ\GammaΓ，可以精准预测或复现在特定延迟下Combo能否成立。

间隔点分析

原理分析

由于玩家的击退基本上可以分为两段

  1. Mi<0M_i < 0Mi​<0, 且逐渐减为0，对击退距离的贡献是负的
  2. Mi>0M_i > 0Mi​>0, 且逐渐增大，对击退距离的贡献是正的

要使得击退距离(DDD)最大，就得使得第一阶段尽量小，第二阶段尽量大，也就是第一阶段尽量大。

间隔点就是为了实现这一目标的，只在击退后点击xxx次，让前半段有点击的0.60.60.6阻力，后半段没有，尽量使得 DDD 最大。
